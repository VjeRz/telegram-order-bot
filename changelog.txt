Version 1.0 – Initial Release (April 4, 2026)
------------------------------------------------
- Basic bot with /start command.
- Reads Order ID from a Google Sheet (columns: Order ID, Channel Name, SalesForce, Tanggal Submit, Status Order, Fallout Reason).
- Replies with order details in plain text.
- Simple conversation handler (no timeout).
- No registration, no approval.
- Direct connection to Telegram API (no proxy).

Version 1.1 – Cloudflare Proxy & Python 3.12 fix (April 4)
----------------------------------------------------------
- Added Cloudflare Worker proxy to bypass Railway IP blocks.
- Added runtime.txt to force Python 3.12.
- Environment variable CLOUDFLARE_WORKER_URL.

Version 1.2 – Case‑insensitive matching & cleaning (April 5)
-------------------------------------------------------------
- clean_text() function removes invisible characters.
- Order ID matching made case‑insensitive (.lower()).
- Removed Markdown parsing to avoid errors.
- Greeting changed to "Semangat Pagi, Masukan Order ID..." (fixed, no time check).

Version 1.3 – Added STO & WOK to order reply (April 6)
-------------------------------------------------------
- Order sheet now expects columns STO and WOK.
- Reply shows 📠 STO and 🪪 WOK.

Version 2.0 – Registration & Approval System (April 10)
--------------------------------------------------------
- New command /register with multi‑step conversation.
- Role groups: Agency, Branch, Technician.
- Sub‑roles: Team Leader, Salesforce, Inputters, Supervisor, Manager, IT, Technician, HSA.
- User data stored in "Bot_Data" spreadsheet (tabs: Users, UsageLog).
- Branch users auto‑approved; Agency & Technician require IT approval.
- IT approval by inline buttons (notifications to IT_TELEGRAM_ID).
- /pending command (IT only) lists pending registrations.
- Every order lookup is logged to UsageLog.
- /report command for usage reports (day/week/month/custom range, CSV for monthly/custom).

Version 2.1 – WOK and SFID only for Agency (April 11)
------------------------------------------------------
- During registration, only Agency group is asked for WOK (4 areas: Manado‑Talaud, Bitung‑Minahasa, Bolaang Mongondow, Gorontalo) and SFID.

Version 2.2 – Fixed approval callback (April 14)
-------------------------------------------------
- Changed users_sheet.update() to update_cell() to fix API error.
- /pending buttons now include row index for correct row update.

Version 2.3 – Welcome guides /guide command (April 14)
-------------------------------------------------------
- New command /guide – tailored welcome/help message based on role.
- After approval, user automatically receives welcome message.
- /start shows registration guide for unregistered users, otherwise directly asks for Order ID.

Version 2.4 – Sales performance report (April 25)
--------------------------------------------------
- New command /salesreport (interactive menu).
- Only Supervisor, Team Leader, IT, Manager can use it.
- Step 1: choose WOK. Step 2: choose Channel (B2B2C&OTHERS, AGENCY, GRAPARI, SOBI AFFILIATE, WEB&APP). Team Leader forced to AGENCY.
- Step 3: choose Month. Counts orders based on Tanggal Input.
- For AGENCY channel: per‑salesperson breakdown with total orders and status counts. Team Leader additionally sees list of Order IDs.
- Messages split to avoid 4096 char limit.

Version 2.13 – Popular paket in Summary report (May 1, 2026)
------------------------------------------------------------
- Summary report (Ringkasan) now sends 5 additional messages:
  * 1 global table: all completed packages across all channels and WOKs, with counts and percentages.
  * 4 per‑WOK tables: completed packages for each working area (MANADO TALAUD, BOLAANG MONGONDOW, GORONTALO - PAHUWATO, BITUNG MINAHASA).
- No top‑N limit – every package with at least one COMPLETED order appears.
- Wording improved: "Cmpl" → "Comp", "Flt" → "FO" in all summary tables.

Version 2.14 – Grapari Performance (per STO) report (May 1, 2026)
-----------------------------------------------------------------
- New report available for Team Leader Grapari and CS Grapari: "📊 Grapari Performance (per STO)".
- User selects year (2025 or 2026) → month → then two options:
  * 📊 Lihat Ringkasan – summary table by STO (city/area) with metrics: Input, Comp, FO, IO/PS, Kontrib, MoM.
  * 📥 Download CSV – export of all orders (any status) for the chosen month, GRAPARI channel only.
- Fixed a bug where the report got stuck after year selection (now deletes old messages and sends fresh ones).

Version 2.15 – Bulk order separator & timestamp footer (May 2, 2026)
--------------------------------------------------------------------
- Bulk order now accepts commas as separators (e.g., AOi123, AOi456, AOi789). Spaces and newlines still work.
- Last update footer now includes time (HH:MM) – e.g., "📅 Last Update Data: 02/05/2026 14:30".
- Improved date parsing to support multiple input formats (DD/MM/YYYY HH:MM, YYYY-MM-DD HH:MM:SS, etc.).
- Minor stability fixes for menu display from callback queries.

================================================================================
                              CURRENT FEATURES
================================================================================

- Registration with role selection (Agency, Branch, Technician, Grapari). Agency users also enter WOK and SFID.
- IT approval via /pending with inline buttons.
- Single order lookup (🔍 Cek Order) – shows all fields including Paket.
- Bulk order lookup (2–10 IDs) – accepts spaces, commas, or newlines.
- Sales reports:
  * Laporan Detail (per WOK): for Manager/Supervisor/IT shows status breakdown with percentages; for Team Leader (Agency & Grapari) shows CSV download, per‑salesperson list, and popular paket.
  * Ringkasan (per WOK & Channel): 6 tables + 5 popular paket tables (global + per WOK).
  * Grapari Performance (per STO): summary by STO + CSV export (Grapari only).
- Usage report via button (daily/weekly text, monthly CSV).
- Role‑based main menu and guide text (/guide).
- Last update date & time footer in error messages, detail reports, and all summary tables.
- Cloudflare proxy for stable connection.

================================================================================
                              KNOWN LIMITATIONS
================================================================================

- Order sheet >10,000 rows may slow responses; archive old data regularly.
- Sales report processes all rows; acceptable up to ~15k rows (cached every 5 minutes).
- Bulk order IDs must be between 2 and 10.
