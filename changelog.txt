Version 1.0 – Initial Release (April 4, 2026)
---------------------------------------------
- Basic bot with /start command.
- Reads Order ID from a Google Sheet (column headers: Order ID, Channel Name, SalesForce, Tanggal Submit, Status Order, Fallout Reason).
- Replies with order details in plain text.
- Simple conversation handler (no timeout).
- No registration, no approval.
- Direct connection to Telegram API (no proxy).

Version 1.1 – Cloudflare Proxy & Python 3.12 fix (April 4)
----------------------------------------------------------
- Added Cloudflare Worker proxy to bypass Railway IP blocks.
- Added runtime.txt to force Python 3.12 (avoided a library error).
- Environment variable CLOUDFLARE_WORKER_URL.

Version 1.2 – Case‑insensitive matching & cleaning (April 5)
-------------------------------------------------------------
- clean_text() function removes invisible characters (zero‑width spaces, non‑breaking spaces).
- Order ID matching made case‑insensitive (.lower()).
- Removed Markdown parsing to avoid errors.
- Greeting changed to "Semangat Pagi, Masukan Order ID..." (fixed, no time check).

Version 1.3 – Added STO & WOK to order reply (April 6)
-------------------------------------------------------
- Order sheet now expects columns STO and WOK.
- Reply shows 📠 STO and 🪪 WOK.

Version 2.0 – Registration & Approval System (April 10)
--------------------------------------------------------
- New command /register with multi‑step conversation.
- Role groups: Agency, Branch, Technician.
- Sub‑roles: Team Leader, Salesforce, Inputters, Supervisor, Manager, IT, Technician, HSA.
- User data stored in "Bot_Data" spreadsheet (tabs: Users, UsageLog).
- Branch users auto‑approved; Agency & Technician require IT approval.
- IT approval by inline buttons (notifications to IT_TELEGRAM_ID).
- /pending command (IT only) lists pending registrations.
- Every order lookup is logged to UsageLog.
- /report command for usage reports (day/week/month/custom range, CSV for monthly/custom).
- Report access: Manager, Supervisor, HSA, IT.

Version 2.1 – WOK and SFID only for Agency (April 11)
-----------------------------------------------------
- During registration, only Agency group is asked for WOK (4 areas: Manado‑Talaud, Bitung‑Minahasa, Bolaang Mongondow, Gorontalo) and SFID.
- Branch and Technician skip these fields.

Version 2.2 – Fixed approval callback (April 14)
------------------------------------------------
- Changed users_sheet.update() to update_cell() to fix API error.
- /pending buttons now include row index for correct row update.

Version 2.3 – Welcome guides /guide command (April 14)
-------------------------------------------------------
- New command /guide – tailored welcome/help message based on role.
- After approval, user automatically receives welcome message.
- /start shows registration guide for unregistered users, otherwise directly asks for Order ID (no extra guide).

Version 2.4 – Sales performance report (April 25)
-------------------------------------------------
- New command /salesreport (interactive menu).
- Only Supervisor, Team Leader, IT, Manager can use it.
- Step 1: choose WOK (4 options).
- Step 2: choose Channel (B2B2C&OTHERS, AGENCY, GRAPARI, SOBI AFFILIATE, WEB&APP).
   – Team Leader automatically forced to AGENCY (skips channel step).
- Step 3: choose Month (January–December).
- Report counts orders based on Tanggal Input (supports both DD/MM/YYYY and YYYY-MM-DD formats).
- For AGENCY channel:
   – Per‑salesperson breakdown with total orders and status counts (all 27 statuses + OTHER).
   – Team Leader additionally sees a list of Order IDs with their status.
- For non‑AGENCY channels: aggregated totals per status.
- Messages split into separate messages per salesperson to avoid Telegram 4096 character limit.
- No Markdown in reports; plain text with emojis.

Version 2.5 – Full status list & bug fixes (April 25)
------------------------------------------------------
- Predefined list of 27 order statuses (from business requirements).
- Any unknown status counts as "OTHER".
- Fixed gspread exception (CellNotFound does not exist; replaced with generic Exception).
- Removed all parse_mode="Markdown" to avoid "Can't parse entities" error caused by special characters in Order IDs.
- Fixed sales report not showing any output (Message_too_long error) by sending each salesperson separately.
- Added chunking for Team Leader's order list (splits into multiple messages if needed).
- Order lookup restored (was broken by gspread exception).

Current Features (as of April 25, 2026)
----------------------------------------
- Registration with role selection, WOK/SFID for Agency only.
- IT approval via /pending or inline buttons.
- Order lookup (/start) for approved users.
- Usage logging & /report for Manager/Supervisor/HSA/IT.
- Sales performance report (/salesreport) for Supervisor, Team Leader, IT, Manager.
- Help (/help) and Guide (/guide) commands.
- Cloudflare proxy for Railway connectivity.
- Google Sheets integration (order sheet + Bot_Data with Users and UsageLog tabs).
- All messages use plain text (emojis allowed, no Markdown).

Known limitations
-----------------
- Order sheet with >10,000 rows may cause slower responses. Archive old data recommended.
- Sales report processes all rows; acceptable for up to ~10,000 rows. Cache refreshes every 5 minutes.
